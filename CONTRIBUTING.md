# Contributing to LSA Website

Thanks for helping improve the LSA Website.

## Prerequisites

- `Node.js` (LTS recommended)
- `npm` (comes with Node.js)
- Git and a GitHub account

## Getting Started

1. Fork this repository on GitHub.
2. Clone your fork:
   - `git clone https://github.com/<your-username>/lsa-website.git`
3. Move into the project folder:
   - `cd lsa-website`
4. Install dependencies:
   - `npm install`

## Running the Project

- Start the development server:
  - `npm run dev`
- Create a production build:
  - `npm run build`
- Preview the production build locally:
  - `npm run preview`

## Code Quality Checks

- Run lint checks before opening a PR:
  - `npm run lint`

## How to Contribute

1. Create a new branch from `main`:
   - `git checkout -b feat/short-description`
2. Make your changes in small, focused commits.
3. Verify the project still works:
   - `npm run lint`
   - `npm run build`
4. Push your branch to your fork.
5. Open a Pull Request with:
   - A clear title
   - A short description of what changed and why
   - Screenshots for UI changes (if applicable)
6. Ensure code can be easily rediable and expandable
7. Keep often changed data in a config.jsx file or adapt it to a google sheet

## Coding Expectations

- Keep changes scoped to one purpose per PR.
- Follow the existing code style and folder structure.
- Write clear, readable React and JavaScript.

## Knowledge Recommended for Contributors

- HTML
- CSS
- JavaScript
- Basic React
